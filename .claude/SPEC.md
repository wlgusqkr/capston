# 자취맵 SPEC

서울 자취생 · 자취 준비생을 위한 공공데이터 기반 동네 대시보드.

이 문서는 현재 프로젝트의 단일 기획 진실(source of truth)이다. 모든 에이전트는 작업 전 이 문서의 관련 섹션을 읽는다.

---

## 0. 절대 규칙 (모든 에이전트 공통)

이 절은 다른 어떤 규칙보다 우선한다. 위배 시 작업 중단하고 사용자에게 보고.

1. **맵뷰(`/`, `routes/MainMap.tsx` 및 `components/Map/*`)는 히트맵 레이어 옵션 확장을 제외하고 동결.** 다른 모든 동작·레이아웃·스타일은 변경 금지. 히트맵 레이어 확장은 Phase 2 후반에 별도 작업으로 진행하며, 시작 전 사용자 사전 승인 필수.
2. **DB 스키마는 건드리지 않는다.** 새 컬럼 / 새 테이블 / 새 인덱스가 필요하다고 판단되면 즉시 작업 중단하고 사용자에게 요청한다.
3. **참조 전용 데이터 접근.** 백엔드는 기존 테이블에서 SELECT만 한다. 집계 결과 캐시용 마테리얼라이즈드 뷰가 필요하다면 사용자에게 요청.
4. **권한 밖 작업은 위임 또는 사용자 질의.** 자기 에이전트 역할 밖이면 알맞은 에이전트로 위임하거나 사용자에게 물어본다. 추측 진행 금지.

## 1. 프로젝트 개요

### 1.1 이름과 도메인

- 서비스명: **자취맵**
- 대상: 서울에서 자취 중이거나 자취를 준비하는 **대학생 · 사회초년생**
- 기본 공간 단위: **서울 행정동 426개**
- 마감: 2026.06.05

### 1.2 가치 제안

공공데이터를 동 단위로 가공·시각화하여, 사용자가 "이 동네에서 자취하기 괜찮을까?"를 빠르게 판단하도록 돕는다.

차별점:
- 자취생 관점의 파생 지표 (자취촌 지수, 인구 대비 편의시설 밀도, 시간대 혼잡도)
- **대학생이 직관적으로 이해할 수 있는** 풍부한 시각화 (전문가용 대시보드 아님)
- 사용자 입력 데이터(거주 동, 학교, 리뷰)와 공공데이터 결합
- 자연어 질의가 가능한 AI 에이전트 (Phase 4)

### 1.3 디자인·인터랙션 톤

**전문가용 대시보드가 아니다.** 그라파나·기상청 대시보드처럼 출처와 정확값을 명시하되, **느낌은 가볍고 친근**해야 한다.

- **레이아웃**: 위젯이 화면에 **플로팅된 느낌**으로 배치. 카드 간 여백 충분히, 라운드 코너, 부드러운 그림자 또는 톤다운 배경.
- **컬러**: 알록달록하되 과하지 않게. 각 데이터 카테고리(부동산/교통/편의시설/안전 등)마다 고유 컬러 계열.
- **인터랙션**:
  - 게이지는 0에서 목표값까지 차오르는 애니메이션
  - 막대·도넛은 마운트 시 길이/각도가 부드럽게 자라남
  - 라인은 좌→우로 그려짐
  - 숫자 카운트업 (예: 0 → 70만)
  - 호버 시 툴팁 + 미세 확대
  - 스크롤 진입 시 위젯 fade-in + slight slide-up
- **귀여운 디테일**: 빈 상태 일러스트 + 짧은 카피, 등락 화살표 색·아이콘, 카테고리별 작은 아이콘.
- **금지**: 차트 색을 회색·검정 위주로 사용, 모서리 직각, 정보 밀도만 추구해서 빽빽하게 채우기.

## 4. 대시보드 화면 (`/dashboard`) — 본 작업의 핵심

### 4.1 목적

선택된 동의 다양한 지표를 한 페이지에서 풍부하게 시각화. 대학생 사용자가 스크롤하면서 "이 동네 어떤 곳이지?"를 즉각 감 잡을 수 있어야 한다.

### 4.2 화면 단위와 동 선택

- 모든 위젯은 **하나의 선택된 동(행정동)** 기준으로 데이터를 보여준다.
- 동 선택 방법:
  - 페이지 상단 동 셀렉터 (검색 가능 콤보)
  - 대시보드 작은 지도에서 동 클릭
- 선택된 동은 URL에 반영 (`/dashboard?dong=<slug>`). 새로고침 후에도 복원.

### 4.3 레이아웃

세로 스크롤. 좌측 패널 없음. 위젯은 플로팅 카드로 그리드 위에 배치.

```
┌──────────────────────────────────────────────────────────────────────┐
│ TopNav  [자취맵]  [맵·대시보드]  [✨AI 검색]              [user]    │
├──────────────────────────────────────────────────────────────────────┤
│ 대시보드 헤더 — 동 셀렉터 + 한 줄 요약 (구·면적·인구)               │
├──────────────────────────────────┬───────────────────────────────────┤
│ KPI 위젯 그리드 (4~6개)          │ 작은 지도 (제1사분면 크기)        │
│ — 큰 숫자 + 등락 + 출처          │ — 필터 칩, 히트맵 토글, 확장 버튼 │
│                                  │   Outlet의 1/4 크기 차지          │
├──────────────────────────────────┴───────────────────────────────────┤
│ 섹션 A. 부동산 시세                                                 │
├──────────────────────────────────────────────────────────────────────┤
│ 섹션 B. 편의시설                                                    │
├──────────────────────────────────────────────────────────────────────┤
│ 섹션 C. 교통                                                        │
├──────────────────────────────────────────────────────────────────────┤
│ 섹션 D. 인구·사회                                                   │
├──────────────────────────────────────────────────────────────────────┤
│ 섹션 E. 안전·환경·경제                                              │
├──────────────────────────────────────────────────────────────────────┤
│ 섹션 F. 인기 차트                                                   │
├──────────────────────────────────────────────────────────────────────┤
│ 섹션 G. 자취생 리뷰                                                 │
└──────────────────────────────────────────────────────────────────────┘
```

**작은 지도 크기**: TopNav를 제외한 Outlet 영역 기준 **가로·세로 각 1/2** = 전체의 **1/4 (제1사분면)**. 화면 폭 1280~1920px 범위에서 비율 유지.

각 위젯이 항상 표시해야 할 것:
- **제목** (간결)
- **현재 값** (큰 숫자 + 단위)
- **출처** (작은 글씨, 예: 국토부 RTMS · 2026-04 기준)
- **자치구 단위 지표는 "○○구 단위" 뱃지** (해당 동이 속한 자치구 값으로 표시)
- 가능 시 비교 (서울 평균 / 자치구 평균 / 전월 대비 등락)

### 4.4 위젯 카탈로그

각 위젯은 (시각화 / 데이터 / 인터랙션) 3요소로 정의된다.

#### 헤더

| 위젯 | 시각화 | 데이터 |
|---|---|---|
| 동 셀렉터 | 검색 가능 콤보박스 (426개) | `useDongScores` |
| 한 줄 요약 | "OO구 OO동 · 면적 X㎢ · 인구 N명" | `adong` + `adong_population` |

#### KPI 행 (4~6개, 모두 카운트업 + 등락 화살표)

| 위젯 | 시각화 | 데이터 | 비고 |
|---|---|---|---|
| 평균 환산 월세 | 큰 숫자 + 미니 라인 + MoM% | `rent_deal` 최근 12개월 | `lib/rent.ts` 재사용 |
| 평균 보증금 | 큰 숫자 + 분포 막대 | `rent_deal` | |
| 계약 건수 | 큰 숫자 + 월별 막대 | `rent_deal` 최근 12개월 | "자취 활발도" |
| 인구 / 가구 수 | 큰 숫자 + 추이 미니라인 | `adong_population` | |
| 안전 지수 게이지 | **차오르는 게이지 (0~100, 약 1.5초)** | `gu_metric.SAFETY_GRADE_MEAN` | **○○구 단위 뱃지** |
| 자취촌 지수 | 게이지 + 상위 % | 파생 (4.5) | |

#### 작은 지도 (제1사분면)

- **크기**: Outlet의 1/4 (가로 1/2 × 세로 1/2)
- 기본: 선택 동 폴리곤 강조
- **플로팅 필터 칩**: 마트 / 약국 / 편의점 / 음식점 / 카페 / 병원 / 공원(대형) / 도서관(공공·작은 색 구분) / 지하철역 / 버스 정류장 / 대학
- 핀 클릭 → 핀 확장 + 이름·간단 정보
- 우상단 확장 버튼 → `/?dong=<slug>` (맵뷰로 이동, 해당 동 강조)
- **히트맵 레이어 토글** (택1):
  - 종합 점수 / 평균 월세 / 계약 활발도 / 청년 비율 / 자취촌 지수 / 안전 지수
- **히트맵 컬러**: 각 레이어마다 다른 컬러 계열 (섹션 4.6 참조)
- **컬러칩 범례**: 작은 지도 우하단, 현재 선택된 레이어의 4~5단계 컬러칩

#### 섹션 A. 부동산 시세

| 위젯 | 시각화 | 데이터 |
|---|---|---|
| 월별 평균 환산 월세 추이 | Recharts 라인 (12~36개월, 좌→우 그려짐) | `rent_deal` |
| 주택 유형 분포 | 도넛 (회전 등장) | `rent_deal.housing_type` |
| 면적 × 환산 월세 산점도 | 산점도 (점들이 통통 나타남) | `rent_deal` |
| 보증금 대역 분포 | 히스토그램 (막대가 자라남) | `rent_deal.deposit` |
| 면적당 시세 vs 서울 평균 | 막대 + 기준선 | `rent_deal` 파생 |
| 신규 / 갱신 비율 | 도넛 | `rent_deal.contract_type` |

링크: "자세한 시세 탐색 →" → `/dong/:slug/explore`

#### 섹션 B. 편의시설

| 위젯 | 시각화 | 데이터 |
|---|---|---|
| 카테고리별 개수 + 정규화 | 표 + 미니바 (개수 · 인구 1만명당 · 서울 평균 대비 %) | `store` + `business_category` + `ksci_category` |
| 자취생 필수시설 충분도 | 칩 그리드 + 등급 뱃지 (코노 / PC방 / 다이소 / 올리브영 / 편의점 / 카페 / 마트) | KSIC 매핑 |
| 충분도 등급 | "부족 / 보통 / 충분" 컬러 뱃지 | 인구 1만명당 → 서울 백분위 |
| 대형 공원 | 카드 리스트 (이름·면적·도보거리) | `park` + `park_adong` (면적 ≥ N㎡) |
| 도서관 | 카드 (공공/작은 색 구분, 평일 운영시간) | `library` + `library_hours` |

#### 섹션 C. 교통

| 위젯 | 시각화 | 데이터 |
|---|---|---|

| 노선·역 수 | 큰 숫자 카드 | `subway_station` |
| 버스 정류장·노선 수 | 큰 숫자 + 미니 바 | `bus_stop` |
| 지하철 시간대 혼잡도 | 라인 (평일/토/일 비교) | `subway_congestion` |
| 버스 시간대 혼잡도 | 라인 (출퇴근/평일/주말) | `bus_congestion` |
| 동 성격 추정 | 텍스트 + 아이콘 ("거주 중심" / "상업 중심" / "유동 인구 많음") | 혼잡도 패턴 분석 |

#### 섹션 D. 인구·사회

| 위젯 | 시각화 | 데이터 |
|---|---|---|

| 사용자 거주자 유형 | 도넛 (대학별 / 학년별) | 사용자 입력 (현재 mock) |

#### 섹션 E. 안전·환경·경제 (모두 ○○구 단위 뱃지)

| 위젯 | 시각화 | 데이터 |
|---|---|---|
| 평균 별점 + 리뷰 수 | 큰 별점 카드 | 사용자 입력 (현재 mock) |
| 리뷰 카드 가로 스크롤 | 카드 (학교/학년/별점/본문) | 사용자 입력 |
| 리뷰 작성 CTA | 버튼 → 모달 | Phase 3 |

### 4.5 파생 지표 정의

대시보드 전용 파생 지표. 백엔드 집계 우선, 무거우면 프론트 계산도 OK.

- **자취촌 지수 (0~100)**:
  - 입력: `rent_deal` 최근 12개월
  - 공식 후보: `0.5 × (비아파트 비율) + 0.3 × (소형 면적 ≤25㎡ 비율) + 0.2 × (월세 계약 건수 정규화)`
  - 출력: 서울 전체 동 백분위
  - 가중치는 초안. 데이터 점검 후 사용자와 조정.
- **계약 활발도**: 최근 12개월 계약 건수 / 인구 → 서울 백분위
- **편의시설 충분도 (카테고리별)**: 점포 수 / (인구 / 1만) → 서울 백분위. <50 "부족", 50~75 "보통", ≥75 "충분"
- **동 성격 추정**: 지하철·버스 시간대 혼잡도 패턴
  - 출퇴근 피크 강 + 낮 저 → "주거 중심"
  - 낮 고른 분포 → "상업·업무 중심"
  - 주말 피크 → "유동인구 많음"

각 파생 지표는 (i) 툴팁으로 정의·공식을 표시.

### 4.6 히트맵 컬러 매핑

작은 지도 및 (장기적으로) 맵뷰의 히트맵 레이어는 종류마다 다른 컬러 계열을 쓴다. 정확한 색은 `design-system-keeper`가 디자인 토큰으로 정의.

| 레이어 | 컬러 계열 | 의미 방향 |
|---|---|---|
| 종합 점수 | 청록 → 진청록 | 낮음 → 높음 |
| 평균 월세 | 옅은 노랑 → 진한 주황 | 저렴 → 비쌈 |
| 계약 활발도 | 옅은 분홍 → 진한 자주 | 한산 → 활발 |
| 청년 비율 | 옅은 보라 → 진한 보라 | 낮음 → 높음 |
| 자취촌 지수 | 옅은 민트 → 진한 청록 | 낮음 → 높음 |
| 안전 지수 | 빨강 → 노랑 → 초록 | 위험 → 안전 |

각 레이어는 4~5단계 양자화. 작은 지도 우하단에 **컬러칩 범례** 표시 (레이어별 다른 범례). 컬러는 디자인 토큰에 카테고리별로 정의: `--color-heatmap-rent-100..500`, `--color-heatmap-safety-low/mid/high` 같은 식.

## 9. 백엔드 변화

### 9.1 원칙

- DB 스키마 변경 금지. 필요 시 사용자에게 요청.
- 기존 hook / endpoint는 가능한 한 재활용.
- 새 endpoint는 동 단위 / 자치구 단위 집계 위주. 무거운 계산은 캐시 (django-redis, 5분 TTL).

### 9.2 기존 hook 활용 매핑

| 기존 hook | 대시보드 활용 |
|---|---|
| `useDongScores` | 동 셀렉터, 인기 차트 후보 |
| `useDongSummary` | KPI 행 일부 |
| `useDongDetail` | 섹션 B~E 다수 |
| `useDongExplore` | 섹션 A 차트 |
| `useTransactions` | 작은 지도 거래 핀 (선택) |
| `useDongMatchCounts` | 계약 활발도 히트맵 |
| `useCompare` | 인기 차트 / 비교 |
| `useFavorites` | 마이페이지 연계 |

### 9.3 신규 endpoint 후보

> 신규 추가 전 백엔드 에이전트가 기존 endpoint 확장으로 충분한지 검토 후 사용자에게 보고.

1. `GET /api/dongs/:slug/dashboard-summary` — 헤더 + KPI 묶음
2. `GET /api/dongs/:slug/transit-congestion` — 시간대 혼잡도 패턴
3. `GET /api/dongs/:slug/derived-indices` — 자취촌 지수, 계약 활발도, 충분도 등급
4. `GET /api/dongs/:slug/amenity-counts` — 카테고리별 개수 + 정규화 + 백분위
5. `GET /api/dongs/ranking?metric=...&university=...&limit=10` — 인기 차트
6. (AI) `POST /api/ai/chat` — 향후 LLM 연결. **현재는 미구현, 인터페이스만 stub**

리뷰·사용자 거주 데이터 관련은 DB 스키마 결정 후 별도 정의.

### 9.4 사용자 입력 데이터 처리

리뷰·거주자 분포·인기 차트는 사용자 데이터에 의존. 데이터 없을 때:
- 위젯은 "데이터 수집 중" 빈 상태 (귀여운 일러스트 + 짧은 카피)
- API 응답 스키마는 미리 정의, 빈 배열·null 반환

## 13. 검증 기준

- 대시보드 페이지가 `/dashboard?dong=<slug>` 로딩 시 Phase 1 위젯이 출처·기준시점·단위·자치구 뱃지(해당 시)와 함께 렌더된다.
- 동 셀렉터 변경 시 모든 위젯이 새 동 데이터로 갱신된다.
- 새로고침 후에도 선택 동이 유지된다 (URL 기반).
- TopNav AI 검색창 클릭 시 사이드 패널이 우측에서 슬라이드 인되고, 닫기로 부드럽게 닫힌다.
- 작은 지도가 Outlet의 1/4 크기를 차지하고, 히트맵 토글 변경 시 컬러 계열과 컬러칩 범례가 함께 바뀐다.
- 차트는 마운트 시 애니메이션이 실행되고, 게이지는 0에서 목표값까지 차오른다.
- 맵뷰(`/`)는 Phase 2 후반 작업 전까지 동작이 완전히 동일하다 (`git diff` 상 변경 없음).
- DB 스키마 변경 없음.
