## 3. 핵심 어필 포인트

### 3.1 점수 계산
3개 sub-score를 백분위로 정규화 → 사용자 가중치로 합산.

```
score_total = score_rent · w_rent + score_amenity · w_amenity + score_transit · w_transit
```

**환산 월세** — 보증금/월세 trade-off 통일:

```
converted = monthly_rent + deposit · 0.005    # 만원/월
                              ↑ 연 6% 전월세전환률 / 12개월
```

이 계수가 점수·차트·비교 표·실거래 핀 hover 모두 동일 (`apps/realestate/utils.py` 단일 진실).

자세한 알고리즘은 [`DATA.md §4`](DATA.md).

### 3.2 선호 학습
5쌍 비교 → 5개 부등식 (선택된 쪽이 점수 높음) → scipy SLSQP로 가중치 추정.

```python
# Bradley-Terry: P(A > B) = σ(score(A) - score(B))
# 목적함수: -log-likelihood, 제약: w 합 = 1, w >= 0
result = minimize(neg_log_likelihood, x0=[0.33,0.33,0.34],
                  constraints={'type':'eq', 'fun': lambda w: sum(w)-1},
                  bounds=[(0,1)]*3, method='SLSQP')
```

**왜 5쌍?** 정보량 vs 사용자 인내. 첫 쌍은 양극단 (월세 저렴+통학 멀음 vs 월세 비쌈+통학 가까움) — 정보량 최대화.

### 3.3 데이터 정직성
비교/상세 페이지의 8개 지표 중 실제 데이터 적재된 항목만 점수에 반영. 나머지(안전 지수·자취생 비율·통학 시간·자취생 평점)는 회색 "데이터 적재 전" 표시.

> 학부 프로젝트지만 거짓 표기보다 정직 표기가 어필 포인트.

## 5. Q&A 예상

### Q1. "5개 더미 동만 있다고 들었는데 풀 적재했나요?"
A. 네. Phase 0a kickoff에서 데모 5개 구로 시작했고, 발표 며칠 전 25개 구 풀 적재 완료. 약 2.5시간 소요. 결과: 27,050 → 123,012 거래 (4.5배).

### Q2. "점수가 한쪽으로 쏠리진 않나요?"
A. 백분위 정규화라 분포는 항상 평균 50, 표준편차 28.9 균등. 공정한 상대 비교가 됨. 단점: 신촌처럼 모든 게 좋은 동도 100점이 안 나옴 (백분위라서).

### Q3. "실거래 좌표가 정확한가요?"
A. 지번 단위로만 VWorld 지오코딩. 같은 지번 안의 여러 매물은 같은 좌표(SPEC 14.2 — 매물 단위 정밀 좌표 금지). 단독·다가구는 응답에 지번이 없어 좌표 null로 적재.

### Q4. "가중치 자동 추정의 정확성?"
A. 5쌍은 사용자 인내 한계. 첫 비교는 양극단으로 정보량 최대화. scipy SLSQP는 제약있는 비선형 최적화의 표준. 결과는 `(w_rent, w_amenity, w_transit)` 합 1.0.

### Q5. "왜 동(洞) 단위? 매물 단위로 보여주면 안 되나요?"
A. (1) 매물 단위면 점수의 의미가 약해짐 — 같은 동 안에서도 매물별 차이가 점수 모델로 잡히지 않음. (2) 사용자는 "이 동네는 어떨까"가 첫 질문이지 "이 매물은"이 아님. (3) 매물은 줌인 시 핀으로 별도 노출.

### Q6. "안전 지수가 비어 있는데?"
A. 적재 전이라 정직하게 회색으로 표시. 향후 경찰청 범죄통계 + 서울 CCTV 위치 조합으로 점수화 예정 ([`ROADMAP.md §3.3`](ROADMAP.md)).

### Q7. "스택 선택 이유?"
A. (자세히는 [`ARCHITECTURE.md §3`](ARCHITECTURE.md)) 요약: GeoDjango = 공간 쿼리 1급 지원. Leaflet = 2D 충분. 단일 VM = 학부 트래픽 적정. 카카오 X = 외부 의존성 회피.

### Q8. "확장 가능성?"
A. 마이크로서비스 X. 단일 모놀리식이 학부 데모에 충분. 트래픽 늘면 (1) PostgreSQL 읽기 복제, (2) Redis 캐시 활성화 (현재 켜짐), (3) CDN 단계적 확장. 자세히 [`ROADMAP.md §4`](ROADMAP.md).

### Q9. "상용 서비스(다방·직방)와 차이?"
A. 다방/직방은 매물 검색 → 본격 임장 단계. 우리는 그 **이전 단계** — "어느 동네를 봐야 할지" 좁히는 도구. 보완재.

### Q10. "데이터 갱신 주기?"
A. 월 1회 cron 권장 (국토부 실거래 갱신 주기). GitHub Actions로 스케줄 가능. 현재 수동.
