#### N1. ~~`Score.css:13` 하드코딩 28px~~ — 수정 완료
- **위치**: `frontend/src/components/ui/Score.css:13`
- **문제**: `font-size: 28px` 직접 박힘. `--font-h1-size: 28px` 토큰이 이미 존재하는데 미사용.
- **명세 근거**: SPEC 4.3 "H1 - 28px / 600", SPEC 4.4 "Score: 큰 숫자 (28~40px)".
- **조치**: `var(--font-h1-size)`로 교체했음 (이 리뷰 중 메인 코디네이터가 직접 수정).

#### N2. `lib/colors.ts` 색 hex 중복 — 의도적, 보류
- **위치**: `lib/colors.ts:18-29`
- **상황**: `--color-data-low/mid1/mid2/high` 토큰의 hex가 `colors.ts`에 다시 박혀 있음 (라이트/다크 두 세트).
- **이유**: Leaflet 폴리곤 `fillColor`는 CSS 변수를 직접 못 받음. 호버 시마다 `getComputedStyle()` 호출하면 426개 폴리곤에서 비용 큼.
- **결정**: 보류. 4단계 frontend-engineer가 실제 성능 측정 후 필요 시 통합. 이중 출처가 되었지만 토큰 변경 시 둘 다 수정해야 한다는 코멘트가 `colors.ts`에 추가되어 있는지 다음 단계에서 확인.

#### N3. Slider thumb/track px 하드코딩 — 의도적
- **위치**: `Slider.css:35-101` (thumb 18px, track 4px 등)
- **이유**: 토큰 스케일에 없는 정밀 크기. Slider 컨트롤 픽셀 미세 조정.
