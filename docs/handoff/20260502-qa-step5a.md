```json
{
  "slug": "pildong",
  "name": "필동",
  "gu": "중구",
  "score": 60.3,
  "summary": "교통 좋고 생활시설 부족, 자취 입문자에게 추천",
  "rent_avg": 85,
  "nearest_station": {"name": "충무로", "line": "4호선", "walking_min": 8},
  "amenity_level": "normal",
  "single_household_pct": 42.0,

```

- safety_level: transit 90 → "high" ✓

### Notes (트래킹)

#### N1. `nearest_station.walking_min` 직선 거리/속도 가정
- **위치**: `serializers.py:24-30` 더미 매핑.
- **상황**: 현재는 slug별 하드코딩 5건. 실 데이터(SubwayStation 모델)는 10단계.
- **결정**: 허용. 메인 동선 데모 가능.

#### N2. `safety_level` = transit 점수 매핑 (임시)
- **위치**: `serializers.py:162-168`
- **상황**: SPEC에는 별도 안전 지수가 있지만 데이터 부재.
- **결정**: 5/24 이후(SPEC 7번 우선순위 5~6) 정밀화.

#### N3. 한 줄 요약 8단계 우선순위
- ✅ "셋 다 좋음" → "나쁜 조합" → "강한 조합" → "한쪽 강조" → "두 지표 결합" → "한쪽 강조 약" → "균형" → "기본값"
- 우선순위 명확하고 5개 더미 동 모두 fallback "이 동네는 점수 기준상 평범한 편입니다" 미도달 — 룰 커버리지 충분.
