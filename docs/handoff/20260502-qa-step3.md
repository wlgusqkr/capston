### Notes (다음 단계 트래킹)

#### N1. Dong.slug `max_length=80` (SPEC 미명시)
- **위치**: `models.py:15`
- **상황**: SPEC 섹션 10은 `SlugField(unique=True)`만 명시 (max_length 없음). 우리는 80 사용.
- **이유**: 한국어 행정동을 영문 slugify하면 길어질 수 있음 + code 끝 4자리 붙이기.
- **결정**: 허용. 영향 없음.

#### N2. Dong에 `created_at`, `updated_at` 추가 (SPEC에 없음)
- **위치**: `models.py:37-38`
- **상황**: SPEC 10에는 없는 필드.
- **이유**: 디버깅 + 데이터 갱신 추적용. 필수까진 아님.
- **결정**: 허용. 응답에 노출되지 않음(`fields` 명시).

#### N3. GeoJSON 부재 (1단계부터 알려진 이슈)
- **상황**: 더미 5개로 데모. 메인 지도는 폴리곤 렌더링이 핵심인데, 더미 폴리곤은 작은 사각형(약 500m).
- **결정**: 4단계 frontend-engineer가 더미 폴리곤 그대로 사용. 10단계에서 실제 GeoJSON 적재.

#### N4. backend-engineer agent stall
- **상황**: management commands 작성 단계에서 600초 무응답으로 자동 종료.
- **조치**: 메인 코디네이터가 동일 사양으로 마무리. 결과 동일.
- **앞으로**: 다음 sub-agent 작업 시 한 번에 너무 큰 prompt를 주는 대신 더 잘게 쪼갤 수 있음 (4단계 프롬프트 작성 시 고려).
